**公式：**

```
SeasonalMult[m] = 1
```

### 2-B  重塑基准 ReshapedBase → 归一化 NormBase

**目的：** 保留冷暖季波动形状，同时把全年总量「归一化」回 Base_Annual。

**公式：**
```
ReshapedBase[m] = Base_Annual × BaseMonthlyShare[m] × SeasonalMult[m]
NormBase[m]     = ReshapedBase[m] / SUM(ReshapedBase) × Base_Annual
```

**代入示例（以 Jan 为例）：**
```
ReshapedBase[Jan] = 7,778.00 × 0.08548 × 1.050
              = 698.1066 kWh
NormBase[Jan]     = 698.1066 / 8,092.6847 × 7,778.00
              = 670.9607 kWh
```

→ SUM(ReshapedBase) = **8,092.68 kWh**（季节膨胀后，下一步拉回）
→ SUM(NormBase) = **7,778.00 kWh**（= Base_Annual ✅）

### 2-C  热力额外分配 ThermalAlloc[m]

**目的：** 把 Thermal_Extra = 1,260.00 kWh 按设备冷暖月权重分摊到各月。

- 冷季月: ThermalWeight += BaseShare × CoolMonthMult(1.05)
- 暖季月: ThermalWeight += BaseShare × HeatMonthMult(1.08)

**公式：**
```
ThermalWeight[m] = （冷季贡献 + 暖季贡献）
ThermalAlloc[m]  = ThermalWeight[m] / SUM(ThermalWeight) × Thermal_Extra
```

```
代入示例（Jan，冷季）：
ThermalWeight[Jan] ≈ 0.08975
ThermalAlloc[Jan]  = 0.08975 / 0.66581 × 1,260.00
              = 169.85 kWh
```


### 2-D  EV 额外均分 EVAlloc[m]

**目的：** EV 充电电量平均摊到每个月（不区分季节）。

**公式：**
```

           = 0.00 / 12
           = 0.0000 kWh（每月相同）
```


### 2-E  合成最终月电量 FinalKWh[m] 与 FinalShare[m]

**公式：**
```
FinalKWh[m]  = NormBase[m] + ThermalAlloc[m] + EVAlloc[m]
FinalShare[m] = FinalKWh[m] / Final_Annual
```

## Step 3 — 24 小时分布

在基准曲线上叠加「居住模式修正」和「冷暖尖峰修正」，再叠加 EV 充电。

### 3-A  居住模式修正 OccMult[h]

**规则：** 白天时段（H07–H16）乘以 `daytime_mult`，其余小时 = 1.0。

**公式：**
```
OccMult[h] = daytime_mult   如果 h ∈ DaytimeFlag (H07–H16)
           = 1.0            其他时段
```

- `daytime_mult[Mostly away during the day]` = **0.600**（查表 GLOBAL_occupancy_factors.csv）

| 时段 | 小时范围 | OccMult |
|------|---------|---------|
| 白天 | H07–H16 | **0.600** |
| 夜间/早晨 | H00–H06, H17–H23 | 1.000 |

### 3-B  冷暖尖峰修正 PeakMult[h]

**规则：** 尖峰时段叠加制冷/制热乘数，非尖峰时段 = 1.0。

**公式：**
```
PeakMult[h] = CoolPeakMult^CoolPeakFlag[h] × HeatPeakMult^HeatPeakFlag[h]
            （幂次为 0 时该乘数不生效，= 1）
```

- `CoolPeakMult` = **1.10**（系统参与冷季: 是）
- `HeatPeakMult` = **1.08**（系统参与暖季: 是）

**典型小时示例：**

| 小时 | PeakMult | 说明 |
|------|---------|------|
| H00 | 1.0000 | 普通小时 |
| H06 | 1.0800 | 制热尖峰×1.08 |
| H14 | 1.1000 | 制冷尖峰×1.10 |
| H18 | 1.1880 | 制冷尖峰×1.10 + 制热尖峰×1.08 |

### 3-C  调整 & 归一化 AdjShare → NormShare

**公式：**
```
AdjShare[h]  = BaseHourlyShare[h] × OccMult[h] × PeakMult[h]
NormShare[h] = AdjShare[h] / SUM(AdjShare)   ← 强制 24h 合计 = 1.0
```

**代入示例（以 H07 为例，白天 + Mostly away during the day）：**
```
AdjShare[H07] = 0.03884 × 0.600 × 1.0800
              = 0.02517
NormShare[H07] = 0.02517 / 0.88055
              = 0.02858
```


### 3-D  叠加 EV 充电 & 合成 FinalHourlyShare

**目的：** EV 充电按充电模式分布到各小时，与非 EV 用电合并。

**公式：**
```
NonEV_kWh[h]       = (Base_Annual + Thermal_Extra) / 365 × NormShare[h]
EV_kWh[h]          = (EV_Extra / 365) × EVChargingDist[h][mostly_overnight]
FinalKWhPerDay[h]  = NonEV_kWh[h] + EV_kWh[h]
FinalHourlyShare[h]= FinalKWhPerDay[h] / (Final_Annual / 365)
```

- 日均非EV电量 = (7,778 + 1,260) / 365 = **24.7616 kWh**
- 日均EV电量   = 0 / 365 = **0.0000 kWh**

**代入示例（以 H00 为例）：**
