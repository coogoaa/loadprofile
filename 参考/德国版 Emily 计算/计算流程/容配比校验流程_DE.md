以下是 DE 版本的 容配比校验流程参考

说明：
- 本文档整体框架对齐 AU 版（`容配比校验流程_AU.md`），参数全部替换为德国参数。
- **与 AU 最大的区别**：DE 的电池容量**不是**按"整夜 / 晚高峰"用电需求推导，而是按**配储率**（PV 容量 × 系数）推荐。用电需求仍然计算，仅作为方案对比、后期手动调整的参考。
- 参数来源：`参考/德国版 Emily 计算/德国参数/DE_基础参数.md` 等。

## 1. 计算流程

### 1.1 初选（逆变器目标功率）
```
target_inverter_kw = PV_total_kwp / 1.3    # 目标 SCR = 130%
```

### 1.2 匹配（从标准化规格库选最小规格）
- 方案 A / B：在 [5, 6, 8, 10, 12, 15] 中取 ≥ target_inverter_kw 的最小值。
- 方案 C：在 [5, 6, 8, 10, 12, 15, 18, 20, 22] 中取 ≥ target_inverter_kw 的最小值。
- 上限 24 kW；若 target_inverter_kw > 24，则先尝试把 PV 削减到 24 × 1.5 = 36 kWp（实际会被 25 kWp 硬约束卡住）。

### 1.3 校验与调整
```
SCR = PV_total_kwp / inverter_kw
```

| 情形 | 判定 | 处理 |
| --- | --- | --- |
| Case 1 | SCR ≤ 150% | 通过 |
| Case 2 | SCR > 150% | ① 优先增大逆变器规格；② 若已达本方案最大规格，则触发**削减面板**：`max_pv = inverter × 1.5`，按发电量最小的坡面顺次削减面板直至合规 |

> 说明：DE 三相规格丰富且 PV 受 25 kWp 硬约束限制，Case 2 真正触发的概率很低。逻辑仍需保留以应对极端场景（例如方案 A/B 下 PV≈25、最大规格 15 kW 时 SCR≈167%，需削减至 ≤22.5 kWp）。

### 1.4 电池容量推荐（**DE 专属：配储率法**）
```
Battery_target_kwh = PV_total_kwp × storage_ratio          # A=0.7, B=0.9, C=1.2
Battery_nominal   = ceil_to_spec(Battery_target_kwh,
                                 [5, 6.5, 9.6, 10, 13.5, 16, 20, 25, 30, 35, 40, 45, 50])
Battery_nominal   = clamp(Battery_nominal, 5, 50)
```

- 不使用 aSurplus / bSurplus 光伏剩余联动逻辑；
- 不使用"整夜需求 / 晚高峰需求"作为电池下限；
- DoD、RTE 仅用于后续能量流仿真，不进入电池规格选型。

### 1.5 用电需求计算（保留，用于对比与人工调整）
```
daily_kwh        = annual_kwh / 365
daytime_kwh      = daily_kwh × 43.55%      # 09:00–17:00
evening_peak_kwh = daily_kwh × 16.55%      # 18:00–20:00
nighttime_kwh    = daily_kwh × 56.45%      # 夜间 15h
```

> 后续"电池自耗率、白天自用覆盖度、夜间放电时长"等指标都基于此用电切分生成；但方案电池规格本身**不再**由它驱动。

## 2. 完整推导过程示例

### 示例 1：大屋顶（3D 满铺 14.10 kWp），北威州 (NW)

#### 步骤 1：屋顶坡面信息
- 3D 满铺：30 片 × 0.47 kW = 14.10 kWp
- 有效坡面：4 个（示意）

#### 步骤 2：物理限制分析
- 物理最大容量：30 片
- 对应总功率：14.10 kWp
- 25 kWp 硬约束：未触发

#### 步骤 3：用户用电需求计算（NW）
- 年用电量：3,280 kWh
- 日均用电：3,280 / 365 = 8.99 kWh
- 白天用电：8.99 × 43.55% ≈ 3.91 kWh
- 晚高峰用电：8.99 × 16.55% ≈ 1.49 kWh
- 夜间用电：8.99 × 56.45% ≈ 5.07 kWh

#### 步骤 4：方案 A 推导（target 7.05 kWp）
- 所需面板：ceil(7.05 / 0.47) = **15 片**
- PV：15 × 0.47 = **7.05 kWp**
- 逆变器目标：7.05 / 1.3 = 5.42 kW → 在 [5,6,8,10,12,15] 取 **6 kW**
- 容配比：7.05 / 6 = **117.5%** ≤ 150% ✓
- 电池：7.05 × 0.7 = 4.94 kWh → 向上取 **5 kWh**（触底最低标称 5）

#### 步骤 5：方案 B 推导（target 10.34 kWp，未命中 EV/热泵）
- 所需面板：ceil(10.34 / 0.47) = **22 片**
- PV：22 × 0.47 = **10.34 kWp**
- 逆变器目标：10.34 / 1.3 = 7.95 kW → 在 [5,6,8,10,12,15] 取 **8 kW**
- 容配比：10.34 / 8 = **129.25%** ≤ 150% ✓
- 电池：10.34 × 0.9 = 9.31 kWh → 向上取 **9.6 kWh**

#### 步骤 6：方案 C 推导（target 13.16 kWp，未命中 EV/热泵）
- 所需面板：ceil(13.16 / 0.47) = **28 片**
- PV：28 × 0.47 = **13.16 kWp**
- 逆变器目标：13.16 / 1.3 = 10.12 kW → 在 [5,6,8,10,12,15,18,20,22] 取 **12 kW**
- 容配比：13.16 / 12 = **109.67%** ≤ 150% ✓
- 电池：13.16 × 1.2 = 15.79 kWh → 向上取 **16 kWh**

### 示例 2：小屋顶物理受限（3D 满铺 7.05 kWp），拜仁州 (BY)

#### 步骤 1：屋顶坡面信息
- 3D 满铺：15 片 × 0.47 kW = 7.05 kWp

#### 步骤 2：物理限制分析
- 物理最大容量：15 片 / 7.05 kWp
- 25 kWp 硬约束：未触发

#### 步骤 3：用户用电需求计算（BY）
- 年用电量：3,302 kWh
- 日均用电：9.05 kWh
- 白天 / 晚高峰 / 夜间：3.94 / 1.50 / 5.11 kWh

#### 步骤 4：方案 A（target 7.05 kWp）
- PV：15 × 0.47 = **7.05 kWp**（铺满）
- 逆变器目标：5.42 → 取 **6 kW**
- 容配比：7.05 / 6 = **117.5%** ✓
- 电池：7.05 × 0.7 = 4.94 → **5 kWh**

#### 步骤 5：方案 B（target 10.34 kWp，未命中 EV）
- 屋顶铺满仅 7.05 kWp < 目标 → **降级至铺满**
- PV：**7.05 kWp**
- 逆变器目标：5.42 → 取 **6 kW**
- 容配比：117.5% ✓
- 电池：7.05 × **0.9** = 6.35 → 向上取 **6.5 kWh**
  - 注意：电池系数用该方案的 0.9，而不是 A 的 0.7；PV 虽然被物理上限拉低，方案档位仍是 B。

#### 步骤 6：方案 C（target 13.16 kWp，未命中 EV）
- 屋顶铺满仅 7.05 kWp < 目标 → **降级至铺满**
- PV：**7.05 kWp**
- 逆变器目标：5.42 → 取 **6 kW**
- 容配比：117.5% ✓
- 电池：7.05 × **1.2** = 8.46 → 向上取 **9.6 kWh**

---

### 示例 3：命中 EV / 热泵 的大屋顶（3D 满铺 ≥ 25 kWp），下萨克森州 (NI)

#### 步骤 1：屋顶坡面信息
- 3D 满铺：60 片 × 0.47 = 28.20 kWp
- 受 **25 kWp 硬约束** 封顶

#### 步骤 2：物理 / 硬约束分析
- 面板上限：floor(25 / 0.47) = **53 片**
- 封顶 PV：53 × 0.47 = **24.91 kWp**

#### 步骤 3：用电需求（NI，annual 3,411 kWh）
- 日均 9.35 kWh → 白天 4.07 / 晚高峰 1.55 / 夜间 5.28 kWh

#### 步骤 4：方案 A（target 7.05 kWp）
- 面板：15 → PV 7.05；逆变器 6 kW；SCR 117.5% ✓
- 电池：7.05 × 0.7 = 4.94 → **5 kWh**

#### 步骤 5：方案 B（命中 EV → target 上调至 13.16 kWp）
- 面板：28 → PV **13.16 kWp**
- 逆变器目标：13.16 / 1.3 = 10.12 → 取 **12 kW**
- SCR：109.67% ✓
- 电池：13.16 × 0.9 = 11.84 → 向上取 **13.5 kWh**

#### 步骤 6：方案 C（命中 EV → target 上调至 15.04 kWp）
- 面板：ceil(15.04 / 0.47) = 32 → PV = 32 × 0.47 = **15.04 kWp**
- 逆变器目标：15.04 / 1.3 = 11.57 → 取 **12 kW**
- SCR：15.04 / 12 = **125.3%** ✓
- 电池：15.04 × 1.2 = 18.05 → 向上取 **20 kWh**

> 若极端情形下 SCR > 150%（例如方案 A/B 碰到 PV≈25 kWp 且规格库最大 15 kW）：
> - 先尝试升级逆变器到 15 kW；
> - 若已是最大规格，则 `max_pv = 15 × 1.5 = 22.5 kWp`，按发电量最小坡面削减至面板数 = floor(22.5/0.47)=47 片，PV = 22.09 kWp，SCR = 22.09 / 15 ≈ 147.3% ✓。

## 3. 与 AU 版差异汇总

| 维度 | AU | DE |
| --- | --- | --- |
| 目标容配比 | 180% | **130%** |
| 最大容配比 | 200% | **150%** |
| 相位 | 单相（≤10 kW）/ 三相 | **仅三相**，上限 24 kW |
| 面板功率 | 0.44 kW | **0.47 kW** |
| PV 硬上限 | — | **25 kWp** |
| 电池推导 | max(时段需求, aSurplus / bSurplus × 光伏剩余) / 0.855 | **PV × 配储率**（A 0.7 / B 0.9 / C 1.2），直接向上取标准规格 |
| 电池规格 | 含 5 / 7 / 10 / 13.5 / 16 / 20 / 30 / 40 / 50 | 5, 6.5, 9.6, 10, 13.5, 16, 20, 25, 30, 35, 40, 45, 50 |
| 用电时段 | 整夜 17–07 (14h) / 晚高峰 17–21 (4h) | 白天 09–17 / 晚高峰 18–20 / 夜间剩余，份额见 §0.5 |
| 用电需求的角色 | **驱动**电池容量 | 仅作对比与人工调整参考，**不**驱动电池 |
