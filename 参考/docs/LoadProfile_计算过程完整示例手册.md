## 第二章 月度计算公式——小白级逐步讲解

### 2.1 先搞清楚我们要算什么

我们的目标是：**把一个家庭全年的用电量，合理地分配到 12 个月**。

不同家庭的月度分配是不一样的：
- 有空调的家庭 → 夏天用电多（制冷）
- 有电采暖的家庭 → 冬天用电多（供暖）
- 有电动车的家庭 → 每个月都多一点（充电）

所以我们需要根据用户的实际情况，调整每个月的用电占比。

### 2.2 计算需要哪些输入？

| 输入 | 含义 | 示例值 |
|------|------|--------|
| 基础年电量 | 这个州/国家的平均家庭年用电量 | NSW: 7,778 kWh |
| 月度基准占比 | 没有任何干预时，每个月占全年的比例 | 1月: 8.548% |
| 供暖/制冷系统 | 用户家里有什么冷暖设备 | Heat pump |
| 使用强度 | 冷暖设备用得多不多 | Medium |
| EV 年里程 | 电动车一年开多少公里 | 10,000 km |
| 冷暖季定义 | 哪几个月是冷季、哪几个月是暖季 | NSW: 冷季Dec-Mar, 暖季Jun-Aug |
| 系数表 | 不同强度对应的乘数 | Medium: 冷季月乘数1.10, 暖季月乘数1.15 |
| 热负荷表 | 不同系统的基础热负荷 | Heat pump: 1,800 kWh |

### 2.3 完整示例：澳洲 NSW + Heat pump + Medium + EV 10,000km

#### 用户画像

```

EV 年里程：10,000 km
EV 充电习惯：Mostly overnight（不影响月度，只影响24h）
居住占用模式：Mostly away（不影响月度，只影响24h）
```

#### 查表获取参数

从预设 CSV 中查到：

```

Heat pump 基础热负荷 = 1,800 kWh          ← AU_hvac_thermal_load.csv

Medium 冷季月乘数 (CoolMonthMult) = 1.10  ← AU_usage_level_coefficients.csv

EV 能效 = 0.18 kWh/km                     ← GLOBAL_ev_params.csv

```

---

#### Step 1：计算最终年电量

```
热力额外电量 = 基础热负荷 × 年电量倍率
             = 1,800 × 1.0
             = 1,800 kWh

EV 额外电量 = 里程 × 能效
            = 10,000 × 0.18
            = 1,800 kWh

最终年电量 = 基础年电量 + 热力额外 + EV额外
           = 7,778 + 1,800 + 1,800
           = 11,378 kWh
```

> 解读：这个家庭因为有热泵和电动车，年用电量从 7,778 涨到了 11,378 kWh。

---

#### Step 2：计算每个月的季节乘数

季节乘数的意思是：**冷暖季的月份，用电量要放大一点**。

公式：
```
SeasonalMult[m] = 1
  + IF(该月是冷季, CoolMonthMult - 1, 0)    ← Heat pump 参与冷季
  + IF(该月是暖季, HeatMonthMult - 1, 0)    ← Heat pump 参与暖季
```

因为选的是 Heat pump，它**既参与冷季修正，也参与暖季修正**。

逐月计算：

| 月 | 冷季? | 暖季? | 冷季修正 | 暖季修正 | SeasonalMult |
|----|-------|-------|---------|---------|-------------|
| 1月 | ✅ 是 | ❌ 否 | 1.10-1=0.10 | 0 | 1 + 0.10 + 0 = **1.10** |
| 2月 | ✅ 是 | ❌ 否 | 0.10 | 0 | **1.10** |
| 3月 | ✅ 是 | ❌ 否 | 0.10 | 0 | **1.10** |
| 4月 | ❌ 否 | ❌ 否 | 0 | 0 | **1.00** |
| 5月 | ❌ 否 | ❌ 否 | 0 | 0 | **1.00** |
| 6月 | ❌ 否 | ✅ 是 | 0 | 1.15-1=0.15 | 1 + 0 + 0.15 = **1.15** |
| 7月 | ❌ 否 | ✅ 是 | 0 | 0.15 | **1.15** |
| 8月 | ❌ 否 | ✅ 是 | 0 | 0.15 | **1.15** |
| 9月 | ❌ 否 | ❌ 否 | 0 | 0 | **1.00** |
| 10月 | ❌ 否 | ❌ 否 | 0 | 0 | **1.00** |
| 11月 | ❌ 否 | ❌ 否 | 0 | 0 | **1.00** |
| 12月 | ✅ 是 | ❌ 否 | 0.10 | 0 | **1.10** |

> 解读：冷季月份（夏天 Dec-Mar）乘以 1.10，暖季月份（冬天 Jun-Aug）乘以 1.15，其他月份不变。

| 9月 | 0.07356 | 572.14 | 1.00 | **572.14** |

#### Step 5：热力额外电量按季节权重分配

热力额外电量 = 1,800 kWh，需要分配到各月。

**分配逻辑**：Heat pump 既参与冷季也参与暖季，所以冷暖季月份都有权重。

公式：
```
ThermalWeight[m]:
  IF(m∈冷季): BaseShare[m] × CoolMonthMult = BaseShare[m] × 1.10
  IF(m∈暖季): BaseShare[m] × HeatMonthMult = BaseShare[m] × 1.15
  两者可叠加（如果某月既是冷季又是暖季，但实际不会出现）
  其他月份: 0
```

## 第三章 24 小时计算公式——小白级逐步讲解

### 3.1 我们要算什么

上一章算出了每个月用多少电。这一章要算：**一天 24 小时里，每个小时用多少电**。

不同家庭的 24h 分布也不一样：
- 白天上班不在家 → 白天用电少，晚上用电多
- 居家办公 → 白天用电多
- 有空调 → 下午 14-19 点用电多（制冷高峰）
- 有电采暖 → 早晚 06-09 和 18-22 用电多（供暖高峰）
- 有电动车 → 看充电习惯，可能集中在夜间或白天

### 3.2 完整示例（接上一章的 NSW 用户）

继续用上一章的用户：
```
NSW + Heat pump + Medium + EV 10,000km + Mostly overnight + Mostly away
最终年电量 = 11,378 kWh
热力额外 = 1,800 kWh
EV 额外 = 1,800 kWh
```

查表获取参数：
```

Medium 暖峰乘数 (HeatPeakMult) = 1.15   ← AU_usage_level_coefficients.csv

暖峰时段 = 06:00-09:00 & 18:00-22:00 (H06-H08, H18-H21)
```

| 12月 | ✅ | **1.35** |
| 其他 | ❌ | **1.00** |

夏天月份被放大 35%！归一化后夏天占比显著上升。
EV 额外 4,500/12 = 375 kWh/月均匀叠加。

#### Step 3：24h 分布

WFH：白天 H07-H16 ×1.4
AC Very high 冷峰 H14-H18：×1.50
Solar-optimized 充电：白天按光伏曲线

| 时段 | OccMult | PeakMult | EV | 合成效果 |
|------|---------|---------|-----|---------|
| H00-H05 | 1.0 | 1.0 | 少量夜间(20%) | 低 |
| H06 | 1.0 | 1.0 | 无 | 正常 |
| H07-H08 | 1.4 | 1.0 | 无 | 白天偏高 |
| H09-H13 | 1.4 | 1.0 | 太阳能充电 | **高** |
| H14-H16 | 1.4 | 1.50 | 太阳能充电 | **极高** (×2.1 + EV) |
| H17 | 1.0 | 1.50 | 无 | 高 |
| H18 | 1.0 | 1.50 | 无 | 高 |
| H19-H23 | 1.0 | 1.0 | 少量夜间(20%) | 正常 |

> 结论：下午 14-16 时段是全天绝对高峰（WFH ×1.4 × 冷峰 ×1.50 = ×2.1，再加上太阳能充电）。
> 这个用户的用电曲线非常适合太阳能自发自用——白天用电量极高，正好匹配光伏发电。

## 第五章 不同系统类型的行为对比总结

| 系统 | 增加年电量? | 参与冷季月度修正? | 参与暖季月度修正? | 参与冷峰24h修正? | 参与暖峰24h修正? |
|------|-----------|----------------|----------------|----------------|----------------|
| No system | ❌ | ❌ | ❌ | ❌ | ❌ |
| Air conditioning | ❌ | ✅ | ❌ | ✅ | ❌ |
| Electric heating | ✅ | ❌ | ✅ | ❌ | ✅ |
| Heat pump | ✅ | ✅ | ✅ | ✅ | ✅ |

> Heat pump 是影响最全面的系统——它同时影响年电量、冷暖季月度分布、冷暖峰24h分布。
> Air conditioning 最特殊——它不增加年电量，只改变分布形状。

### 7.3 新增国家需要准备的文件

最少需要 6 个文件：
1. `XX_base_annual_kwh.csv` — 基准年电量
2. `XX_monthly_share.csv` — 月度基准占比
3. `XX_hourly_share.csv` — 24h 基准占比
4. `XX_cooling_heating_season.csv` — 冷暖季定义

6. `XX_usage_level_coefficients.csv` — 使用强度系数（可复用）

通用文件（`GLOBAL_*`）不需要重复配置。
