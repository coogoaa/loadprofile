# SaleAgent 个性化方案推荐 — 产品架构设计文档

> 版本：V2.0 Draft
> 目标：构建一个支持多国扩展、千人千面的太阳能+储能 ChatBot 个性化推荐系统

---

## 一、产品愿景

我们的 ChatBot 不是一个"填表计算器"，而是一个**对话式太阳能顾问**。它通过自然对话逐步收集用户信息，结合用户所在国家/地区的规则，生成个性化的太阳能+储能方案推荐。

核心理念：**千人千面** — 每个用户看到的方案都是基于其真实用电习惯、设备偏好、屋顶条件和财务目标量身定制的。

## 三、用户画像模块详解

用户画像是整个系统的信息中枢。Agent 通过对话逐步收集，每个字段都有"默认值"兜底，信息越多方案越精准。

### 3.1 基础信息

| 字段 | 说明 | 收集方式 | 默认值 | 影响范围 |
|------|------|---------|--------|---------|
| 国家 | AU / DE / ... | 必填 | — | 决定整套参数包 |
| 地区/州 | AU: NSW/VIC/QLD/... DE: 城市 | 地址自动识别或对话 | 按国家首都 | 基准电量、季节、发电系数 |
| 电网相数 | 单相 / 三相 / 不清楚 | 对话询问 | AU: 单相; DE: 三相 | 逆变器规格上限 |
| 电价结构 | 统一电价 / TOU / 阶梯 | 对话询问 | 统一电价 | 投资回报计算 |
| 电费零售商 | 具体零售商名称 | 对话询问（可选） | 不指定 | 精确电价（预留） |
| 税务政策 | 按国家自动适用 | 自动 | 按国家规则 | 报价计算 |

**税务规则预留**：
- 澳洲：GST 10%，补贴按电池容量
- 德国：PV ≤30kW 免税（0% VAT）；仅加电池需缴 19% VAT

### 3.2 用电画像（LoadProfile 因子）

| 字段 | 说明 | 收集方式 | 默认值 | 影响维度 |
|------|------|---------|--------|---------|
| 居住占用模式 | 白天是否在家 | 对话 | Mostly away | 24h 分布 |
| 供暖/制冷系统 | 家中冷暖设备 | 对话 | AU: AC; DE: No system | 年电量+月度+24h |
| 冷暖使用强度 | 使用频率 | 对话（仅当有冷暖系统时） | Medium | 年电量+月度+24h |
| EV 年里程 | 电动车使用情况 | 对话 | No EV | 年电量+月度 |
| EV 充电习惯 | 充电时段偏好 | 对话（仅当有 EV 时） | Overnight | 24h 分布 |
| 电费账单 | 上传账单或电表数据 | 上传（预留） | 不提供 | 覆盖年电量 |
| 智能电表数据 | 精确用电曲线 | 上传（预留） | 不提供 | 覆盖全部分布 |

**年电量来源优先级**（从高到低）：
1. 智能电表数据 → 直接使用实际曲线
2. 电费账单解析 → 提取年电量，分布仍用 LoadProfile
3. 用户手动输入年电量 → 替代基准值，分布用 LoadProfile
4. LoadProfile 因子计算 → 基准值 + 干预因子
5. 国家/州预设默认值 → 兜底

### 3.3 设备与偏好

| 字段 | 说明 | 收集方式 | 默认值 | 影响范围 |
|------|------|---------|--------|---------|
| 品牌偏好 | 高端 / 中端 / 入门 | 对话 | 中端 | 设备选型+报价 |
| 安装需求优先级 | 见下方 | 对话 | 长期回报 | 方案排序+推荐话术 |
| VPP 意向 | 已加入 / 计划加入 / 不考虑 | 对话 | 不考虑 | 电池策略+收益计算 |
| 预算范围 | 大致预算区间 | 对话（可选） | 不限 | 方案筛选 |

**安装需求优先级选项**（预留，可多选排序）：
- 自发自用最大化 → 偏向大 PV + 中等电池
- 长期投资回报 → 偏向最优性价比组合
- 备电需求 → 偏向大电池
- 电价套利 → 偏向大电池 + TOU 优化
- 最小初始投入 → 偏向入门方案

### 3.4 既有系统信息

| 字段 | 说明 | 收集方式 | 默认值 | 影响范围 |
|------|------|---------|--------|---------|
| 系统现状 | 无 / 仅PV / PV+电池 | 对话 | 无 | 决定新建 vs 扩容 |
| 既有 PV 容量 | kW | 对话 | — | 扩容计算 |
| 既有面板数量 | 块 | 对话（可选） | — | 屋顶剩余评估 |
| 既有面板朝向 | 方位角 | 对话（可选） | — | 发电量估算 |
| 安装时间 | 年份 | 对话 | — | 设备老化评估 |
| 品牌型号 | 具体型号 | 对话（可选） | — | 兼容性评估 |
| 既有电池 | 型号+容量 | 对话（仅当有电池时） | — | 扩容策略 |

### 3.5 安装商配置（预留）

| 字段 | 说明 | 来源 |
|------|------|------|
| 可售品牌列表 | 按高端/中端/入门分类 | 安装商后台配置 |
| 具体产品型号 | 面板/逆变器/电池的具体 SKU | 安装商后台配置 |
| 自定义报价 | 覆盖默认单价 | 安装商后台配置 |
| 安装费用 | 额外安装成本 | 安装商后台配置 |

## 四、LoadProfile 计算引擎

### 4.1 设计原则

- **统一引擎，参数驱动**：同一套计算逻辑，通过国家参数包实现差异化
- **三层分离**：年电量、月度占比、24h 占比独立计算后合成
- **归一化保证一致性**：每次修正后重新归一化
- **叠加式设计**：基础负荷 + 热力负荷 + EV 负荷分别计算后叠加
- **向后兼容**：默认因子下的输出与现有预设值一致

### 4.2 计算流程

```
输入：5 大干预因子 + 国家参数包
  │
  ├─ Step 1：确定最终年电量
  │    Final_Annual = Base_Annual + Thermal_Extra + EV_Extra
  │    · Thermal_Extra = BaseThermalLoad[system] × UsageMult[level]
  │    · EV_Extra = Mileage × EV_Efficiency (0.18 kWh/km)
  │
  ├─ Step 2：计算月度分布 (12个月)
  │    对每个月：
  │    · 季节乘数 = 1 + 冷季修正 + 暖季修正
  │    · 重塑基准 → 归一化 → 分配热力额外 → 分配EV额外
  │    · 输出：FinalShare[m], FinalKWh[m]
  │
  ├─ Step 3：计算24h分布 (24小时)
  │    对每个小时：
  │    · 占用模式修正 × 峰时段修正
  │    · 归一化 → 计算非EV日均 → 叠加EV充电
  │    · 输出：FinalHourlyShare[h], FinalKWhPerDay[h]
  │
  └─ Step 4：派生用电需求指标
       · 日均用电 = Final_Annual / 365
       · 白天用电 = 日均 × SUM(hourlyShare[7..16])
       · 晚高峰用电 = 日均 × SUM(hourlyShare[17..20])
       · 整夜用电 = 日均 × SUM(hourlyShare[17..23] + hourlyShare[0..6])
```

### 4.3 年电量来源的多通道设计

```
                    ┌─────────────────┐
                    │  智能电表数据     │ ← 最高优先级
                    │  (直接使用曲线)   │    覆盖年电量+月度+24h
                    └────────┬────────┘
                             │ 无数据时降级
                             ▼
                    ┌─────────────────┐
                    │  电费账单解析     │
                    │  (提取年电量)     │    覆盖年电量，分布用 LoadProfile
                    └────────┬────────┘
                             │ 无数据时降级
                             ▼
                    ┌─────────────────┐
                    │  用户手动输入     │
                    │  (输入年电量)     │    替代基准值，分布用 LoadProfile
                    └────────┬────────┘
                             │ 无输入时降级
                             ▼
                    ┌─────────────────┐
                    │  LoadProfile     │
                    │  因子计算         │    基准值 + 5大因子干预
                    └────────┬────────┘
                             │ 无选择时降级
                             ▼
                    ┌─────────────────┐
                    │  国家/州预设      │ ← 兜底
                    │  默认值           │
                    └─────────────────┘
```

## 五、国家参数包

### 5.1 参数包结构

每个国家（或州/地区）需要配置一套完整的参数包：

```
Country Config Pack
├── 基准负荷曲线
│   ├── baseAnnualKWh          // 默认年电量
│   ├── monthlyShare[12]       // 月度基准占比
│   ├── hourlyShare[24]        // 24h 基准占比
│   ├── coolingSeasonFlags[12] // 冷季月份标志
│   ├── heatingSeasonFlags[12] // 暖季月份标志
│   ├── daytimeFlags[24]       // 白天时段标志
│   ├── coolingPeakFlags[24]   // 冷峰时段标志
│   ├── heatingPeakFlags[24]   // 暖峰时段标志
│   └── evChargingProfiles     // EV 充电分布（4种模式）
│
├── 干预因子系数
│   ├── occupancyFactors       // 占用模式乘数
│   ├── usageLevels            // 使用强度系数族
│   ├── systemTypes            // 系统类型 → 基础热负荷
│   └── evEfficiency           // EV 能效 kWh/km
│
├── 方案规则（澳洲有，德国预留）
│   ├── planTargets            // A/B/C 方案目标
│   ├── inverterSpecs          // 逆变器规格库
│   ├── batterySpecs           // 电池规格库
│   ├── capacityRatioRules     // 容配比规则
│   └── costParams             // 成本参数
│
├── 税务与补贴
│   ├── taxRules               // 税率规则
│   └── subsidyRules           // 补贴规则
│
└── 设备库（预留）
    ├── panelCatalog           // 面板产品库
    ├── inverterCatalog        // 逆变器产品库
    └── batteryCatalog         // 电池产品库
```

### 5.2 澳洲 vs 德国参数对比

| 参数维度 | 澳洲 | 德国 |

## 六、方案推荐引擎（澳洲）

### 6.1 LoadProfile 如何接入现有方案流程

```
V1.5.0 (现有)                          V2.0 (改造后)
─────────────                          ─────────────
州预设固定值 ──→ 日均用电               LoadProfile 输出 ──→ 日均用电
              ──→ 白天用电 (固定43.88%)                  ──→ 白天用电 (动态)
              ──→ 晚高峰 (固定22.1%)                     ──→ 晚高峰 (动态)
              ──→ 整夜 (固定59.6%)                       ──→ 整夜 (动态)
         │                                          │
         ▼                                          ▼
   方案 A/B/C 计算                            方案 A/B/C 计算
   (公式不变)                                 (公式不变，输入变了)
```

关键点：**方案推荐的公式逻辑完全不变**，只是输入数据从固定预设变为 LoadProfile 动态输出。

### 6.2 改动影响矩阵

| 现有模块 | 是否改动 | 说明 |
|---------|---------|------|
| 屋顶潜力评估 (3.1.1.4) | 不改 | 不依赖用电量 |
| 用电需求计算 (3.1.1.5) | **替换数据源** | 从固定预设 → LoadProfile 输出 |
| 逆变器选型 (3.1.1.6) | 不改 | 不依赖用电量 |
| 电池容量推荐 (3.1.1.7) | 不改公式 | 输入的用电需求数据来源变了 |
| 系统报价 (3.1.1.8) | 不改 | 不依赖用电量 |
| 投资回报 (3.1.3) | **需适配** | 自用率需要用新的 24h 分布 |
| 储能扩容 (3.1.2) | **替换数据源** | 同用电需求计算 |

## 七、对话收集策略

### 7.1 信息收集优先级

Agent 不需要一次性收集所有信息。按以下优先级逐步收集：

```
第一轮（必要信息，快速出方案）：
  ① 地址 → 自动识别国家/州 + 触发屋顶评估
  ② 系统现状 → 新建 vs 扩容
  ③ 电网相数 → 逆变器规格约束

第二轮（丰富用电画像）：
  ④ 居住占用模式
  ⑤ 供暖/制冷系统 + 使用强度
  ⑥ 电动车情况 + 充电习惯

第三轮（精细化偏好）：
  ⑦ 安装需求优先级
  ⑧ 品牌偏好
  ⑨ VPP 意向
  ⑩ 电价结构

可选深度信息：
  · 电费账单上传
  · 智能电表数据
  · 既有系统详细信息
```

### 7.2 渐进式方案更新

```
用户提供地址
  → 基于默认 LoadProfile 出初始方案（兜底方案）

用户回答用电习惯
  → LoadProfile 更新 → 方案自动刷新

用户上传电费账单
  → 年电量覆盖 → 方案再次刷新

用户选择品牌偏好
  → 设备选型更新 → 报价刷新
```

每一轮对话都让方案更精准，但任何时候都有可用的方案输出。

## 八、多国扩展设计

### 8.1 新增国家的步骤

要支持一个新国家，只需要：

1. **配置国家参数包**：基准曲线、季节定义、系数表
2. **配置税务/补贴规则**
3. **配置方案规则**（如果该国需要方案推荐）
4. **配置设备库**（如果有本地化设备）

计算引擎本身不需要改动。

### 8.2 已规划的国家参数包

| 国家 | 基准曲线 | 季节定义 | 方案推荐 | 状态 |
|------|---------|---------|---------|------|
| 澳洲 (AU) | ✅ 各州数据已有 | ✅ 已有 | ✅ A/B/C | 改造中 |
| 德国 (DE) | ✅ BDEW H0 | ✅ 已有 | ❌ 本期不做 | LoadProfile 就绪 |
| 荷兰 (NL) | 待配置 | Jun–Aug / Oct–Apr | 待定 | 预留 |
| 美国 (US) | 待配置 | 按州差异大 | 待定 | 预留 |
