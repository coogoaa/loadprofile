未来还会考虑几个因素，请设计的时候一并考虑并预留：
1. 未来支持 Agent 通过对对话的方式收集：
- 用户的品牌偏好：
  - 例如用户倾向于高端品牌或者性价比高的中端品牌或者注重初始投入的低端入门品牌
- 更详细的用电信息输入：
  - 例如引导用户上传了电费账单或者其他电表信息，前期可以先支持电表信息
- 安装商配置的品牌：
  	- 未来支持安装商配置自己的售卖的商品，并按照低端、中端、高端分类，甚至直接使用具体的商品型号，来作为价格成本依据。
- 用本身的安装太阳能的需求：
  - 例如是优先满足自发自用最大化
  - 优先满足长期投资回报
  - 优先满足备电需求
  - 优先满足套利优先
  - 等等。
- 用户当前其他信息的收集：
  - 是否 已经 加入 VPP，或计划加入 VPP
  - 是否是 TOU 电价结构或者阶梯电价架构
  - 甚至具体的电费零售商
- 用户当前太阳能系统现状：
  - 例如用户当前是否已经有既有太阳能系统（只有 PV 还是已经是 PV 加储能电池都有了）

  - 当前如果已经有储能电池了，具体的产品型号、安装时间、电池容量等等。
- 用户当前家庭的用电相数
  - 可通过对话询问的方式，让用户提供家庭是单相电还是三相电还是不清楚。澳洲有对应的校验规则
  - 德国同理，不过德国单相电似乎不是很多？
- 税务和政策方面
  - 德国 PV 30kw 以下免税，要保证税务合规
  - 德国 只加电池有 19% 的税
