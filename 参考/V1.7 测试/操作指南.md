
### 步骤4：验证关键指标

检查报告中的以下内容：

#### ✅ 容量规则正确性
- 初始容量是否正确计算（面板数 × 0.45kW）
- 目标容量是否符合方案B规则：
  - 初始容量 > 15kW → 目标 13.2kW
  - 初始容量 ≤ 15kW → 目标 10.12kW

#### ✅ 面板选择正确性
- 目标面板数 = ⌊目标容量 / 0.45⌋
- 实际容量 = 选中面板数 × 0.45
- 选中的面板是否为发电量最高的N块

#### ✅ 数据完整性
- 项目信息是否完整
- 建筑信息是否正确
- 年发电量是否合理

## 阶段二：能量流模拟与自用率计算测试

### 步骤1：准备系统组成结果

确认上一步生成的 `system_composition_result.json` 文件路径。

例如：
```
../验证报告/系统组成验证_5621_20260210_173938/system_composition_result.json
```

### 步骤2：运行能量流模拟验证

```bash
# 确保在脚本目录
cd "/Users/paulgao/Documents/augment-projects/V1.5 测试 v2/V1.7 测试/脚本"

python3 verify_energy_flow.py \
  --project-id 5621 \
  --data-dir ../测试数据 \
  --system-composition-result ../验证报告/系统组成验证_5621_YYYYMMDD_HHMMSS/system_composition_result.json \
  --annual-consumption 8000 \
  --battery-capacity 0 \
  --output-dir ../验证报告
```

**参数说明**：
- `--project-id`: 项目ID（与阶段一相同）
- `--data-dir`: 测试数据目录
- `--system-composition-result`: 阶段一生成的系统组成结果JSON文件路径
- `--annual-consumption`: 年用电量（kWh），默认8000
- `--battery-capacity`: 电池容量（kWh），默认0
- `--output-dir`: 验证报告输出目录

**预期输出**：
- 显示 "✓ 数据加载完成"
- 显示 "✓ 能量流模拟验证报告已生成到: ../验证报告/能量流模拟验证_5621_YYYYMMDD_HHMMSS/"
- 显示 "验证完成！"

### 步骤3：查看验证报告

```bash
# 进入最新生成的验证报告目录
cd ../验证报告/能量流模拟验证_5621_*

# 查看综合验证报告
cat 00_综合验证报告.md

# 查看详细推导过程
cat 详细推导.md

# 查看JSON结果
cat energy_flow_result.json | python3 -m json.tool
```

### 步骤4：验证关键指标

检查报告中的以下内容：

#### ✅ 数据关联正确性
- 系统配置是否与阶段一一致（面板数、容量、年发电量）
- 发电矩阵是否正确构建（12个月 × 24小时）

#### ✅ 能量流模拟正确性
- 发电侧平衡：总发电量 = 直接自用 + 电池充电 + 上网
- 用电侧平衡：总用电量 = 直接自用 + 电池放电 + 从电网购电

#### ✅ 自用率计算正确性
- 年度自用率 = 总自用电量 / 总发电量
- 月度自用率是否合理（通常在30%-60%之间）

#### ✅ 月度数据合理性
- 12个月的能量流数据是否完整
- 夏季和冬季的发电量差异是否合理

## 常见问题

### Q1: 下载数据失败怎么办？

**检查网络连接**：
```bash
curl https://file.greensketch.ai/marketing/test/debug/5621/request.json
```

**检查项目ID**：
- 确认项目ID是否正确
- 确认数据是否存在于服务器上

### Q2: 验证报告中发现数据异常怎么办？

**检查数据文件**：
```bash
# 查看panel_location.json
python3 -m json.tool ../测试数据/5621/panel_location.json | head -50

# 检查面板数量
python3 -c "import json; data = json.load(open('../测试数据/5621/panel_location.json')); print(f'面板数量: {len(data)}')"
```

**重新下载数据**：
```bash
rm -rf ../测试数据/5621
python3 download_data.py 5621 --output-dir ../测试数据
```

### Q3: 如何修改用电量或电池容量？

修改 `verify_energy_flow.py` 的参数：

```bash
# 年用电量10000kWh，电池容量10kWh
python3 verify_energy_flow.py \
  --project-id 5621 \
  --data-dir ../测试数据 \
  --system-composition-result ../验证报告/系统组成验证_5621_*/system_composition_result.json \
  --annual-consumption 10000 \
  --battery-capacity 10 \
  --output-dir ../验证报告
```

### Q4: 如何批量测试多个项目？

创建批处理脚本：

```bash
#!/bin/bash
# batch_test.sh

PROJECT_IDS="5621 5546 5442"

for project_id in $PROJECT_IDS; do
    echo "处理项目: $project_id"
    
    # 下载数据
    python3 download_data.py $project_id --output-dir ../测试数据
    
    # 系统组成验证
    python3 verify_system_composition.py \
      --project-id $project_id \
      --data-dir ../测试数据 \
      --output-dir ../验证报告
    
    # 找到最新的系统组成结果文件
    COMP_RESULT=$(ls -t ../验证报告/系统组成验证_${project_id}_*/system_composition_result.json | head -1)
    
    # 能量流验证
    python3 verify_energy_flow.py \
      --project-id $project_id \
      --data-dir ../测试数据 \
      --system-composition-result "$COMP_RESULT" \
      --annual-consumption 8000 \
      --battery-capacity 0 \
      --output-dir ../验证报告
    
    echo "项目 $project_id 完成"
    echo "---"
done
```

运行：
```bash
chmod +x batch_test.sh
./batch_test.sh
```
