## 核心功能

### 1. 数据下载工具

**功能**：
- 从远程API下载测试数据
- 支持批量下载多个案例
- 自动创建目录结构
- 生成下载记录

**使用方法**：
```bash
python3 download_data.py 5421 --output-dir ../data
```

### 2. 系统组成计算

**功能**：
- 计算初始PV容量
- 应用方案B的目标容量规则（15kW阈值）
- 选择最优面板组合
- 统计系统信息

**方案B规则**：
- 初始容量 > 15kW → 目标 13.2kW
- 初始容量 ≤ 15kW → 目标 10.12kW

**使用方法**：
```bash
python3 system_composition.py --data-dir ../data --output-dir ../output
```

### 3. 投资回报计算

**功能**：
- 估算系统成本和补贴
- 模拟20年现金流
- 计算IRR、回本期、ROI、NPV
- 考虑面板衰减和电价上涨

**使用方法**：
```bash
python3 roi_calculation.py --data-dir ../output --output-dir ../output --config financial_config.json
```

### 4. 一键运行脚本

**功能**：
- 自动执行完整工作流程
- 彩色输出和进度提示
- 自动生成结果摘要

**使用方法**：
```bash
./run_all.sh 5421
```

## 与 roi_verify.py 的关系

### 复用的逻辑

| 功能 | roi_verify.py | V1.7脚本 |
|------|--------------|---------|
| IRR计算 | `_irr()` | `calculate_irr()` |
| 面板衰减 | `_degradation_factor()` | 内联计算 |
| 电价上涨 | `_tariff_factor()` | 内联计算 |
| 回本期计算 | 在`simulate_roi()`中 | `calculate_payback_period()` |

### 主要差异

| 方面 | roi_verify.py | V1.7脚本 |
|------|--------------|---------|
| 数据源 | 本地文件 | 远程API |
| 容量规则 | 通用规则 | 方案B特定规则 |
| 自用率模拟 | 小时级详细模拟 | 固定比例简化模型 |
| 脚本结构 | 单一脚本 | 三个独立脚本 |

## 输出文件说明

### system_composition.json

```json
{
  "projectInfo": {项目信息},
  "buildingInfo": {建筑信息},
  "systemComposition": {
    "initialPanelCount": 初始面板数,
    "initialCapacityKw": 初始容量,
    "targetCapacityKw": 目标容量,
    "selectedPanelCount": 选中面板数,
    "actualCapacityKw": 实际容量,
    "totalAnnualGenerationKwh": 总年发电量,
    "capacityUtilization": 容量利用率
  },
  "selectedPanels": [选中的面板列表]
}
```

## 配置参数

### 默认财务参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| buyPricePerKwh | 0.35 | 零售电价 ($/kWh) |
| feedInTariffPerKwh | 0.08 | 上网电价 ($/kWh) |
| priceInflationRate | 0.03 | 电价年增长率 (3%) |
| panelDegradationYear1 | 0.02 | 第一年衰减率 (2%) |
| panelDegradationAfterYear1 | 0.005 | 后续年份衰减率 (0.5%) |
| selfConsumptionRate | 0.4 | 自用比例 (40%) |
| costPerKw | 1500.0 | 每kW系统成本 ($) |
| subsidyPerKw | 1000.0 | 每kW补贴 ($) |
| years | 20 | 计算年限 |

### 自定义配置

修改 `financial_config.json` 来调整参数。

## 已知限制

1. **简化的自用率模型**：使用固定40%自用比例，未考虑小时级用电模式
2. **统一的成本参数**：未区分不同州的成本差异
3. **简化的补贴计算**：未实现各州的具体补贴政策
4. **无电池优化**：当前版本不包含电池容量优化
