## 1. 总览

### 1.1 旧规则（V1.5.0，澳洲）

```
用户输入：地址（自动识别州）
    ↓
查固定预设表（按州）
    ├── 基准年电量（固定值）
    ├── 白天用电占比（固定 43.88%）
    ├── 晚高峰用电占比（固定 22.1%）
    └── 整夜用电占比（固定 59.6%）
    ↓
方案 A/B/C 计算（电池容量推荐、投资回报）
```

**特征**：所有家庭用同一套占比，无个性化，无用电行为建模。

### 1.2 新规则（V1.6.0，引入 LoadProfile）

```
用户输入：地址 + 5大干预因子
    ↓
LoadProfile 计算引擎
    ├── Step 1：确定最终年电量（基础 + 热力额外 + EV额外）
    ├── Step 2：计算月度分布（12个月动态占比）
    ├── Step 3：计算24h分布（24小时动态占比）
    └── Step 4：派生三大用电需求（白天/晚高峰/整夜）
    ↓
方案 A/B/C 计算（公式不变，输入数据变了）
```

**特征**：每个家庭根据实际用电习惯得到个性化曲线，影响所有下游计算。

### 1.3 变化总览

| 模块 | V1.5.0 | V1.6.0 | 是否改动 |
|------|--------|--------|---------|
| 年电量来源 | 州固定预设 | 基准 + 热力 + EV 叠加 | **改动** |
| 月度分布 | 无（不使用） | 12个月动态占比 | **新增** |
| 24h分布 | 州固定占比 | 24h动态占比（按用电习惯） | **改动** |
| 白天用电 | 日均 × 43.88%（固定） | 日均 × 动态占比（H07-H16） | **改动** |
| 晚高峰用电 | 日均 × 22.1%（固定） | 日均 × 动态占比（H17-H20） | **改动** |
| 整夜用电 | 日均 × 59.6%（固定） | 日均 × 动态占比（H17-H06） | **改动** |
| 电池容量推荐公式 | 不变 | 不变 | **不改** |
| 逆变器选型 | 不变 | 不变 | **不改** |
| PV 方案生成 | 不变 | 不变 | **不改** |
| 投资回报自用率 | 固定占比推算 | 动态24h分布推算 | **需适配** |

## 2. Step 1：年电量计算的变化

### 2.1 旧规则

年电量 = 州预设固定值，用户无法干预。

| 州 | V1.5.0 固定年电量 |
|----|-----------------|
| NSW | 7,778 kWh |
| VIC | 6,778 kWh |
| QLD | 7,270 kWh |
| SA | 7,129 kWh |
| WA | 7,634 kWh |
| ACT | 8,632 kWh |
| TAS | 10,148 kWh |
| NT | 10,008 kWh |

### 2.2 新规则

```
最终年电量 = 基础年电量 + 热力额外电量 + EV额外电量

热力额外电量 = 系统基础热负荷 × 使用强度倍率
  （仅 Electric heating 或 Heat pump 时 > 0；AC 和 No system = 0）

EV额外电量 = 年里程 × 0.18 kWh/km
```

**澳洲各系统基础热负荷（新增参数）：**

| 系统类型 | 澳洲基础热负荷 | 德国基础热负荷 |
|---------|------------|------------|
| No system | 0 | 0 |
| Air conditioning | 0 | 0 |
| Electric heating | 2,500 kWh | 3,000 kWh |
| Heat pump | 1,800 kWh | 2,000 kWh |

**使用强度倍率（两国通用）：**

| 强度 | 年电量倍率 |
|------|---------|
| Low | 0.7 |
| Medium | 1.0 |
| High | 1.3 |
| Very high | 1.6 |

### 2.3 对比示例：NSW 家庭

**场景 A — 全默认（无 HVAC，无 EV）：**
```
旧：年电量 = 7,778 kWh
新：年电量 = 7,778 + 0 + 0 = 7,778 kWh  ← 完全一致（向后兼容）
```

**场景 B — 热泵 Medium + 10,000 km EV：**
```
旧：年电量 = 7,778 kWh（固定，无法体现）
新：年电量 = 7,778 + (1,800 × 1.0) + (10,000 × 0.18)
          = 7,778 + 1,800 + 1,800
          = 11,378 kWh  ← 比旧值高 46.3%！
```

**场景 C — 电采暖 High + 无 EV：**
```
旧：年电量 = 7,778 kWh
新：年电量 = 7,778 + (2,500 × 1.3) + 0
          = 7,778 + 3,250
          = 11,028 kWh  ← 高 41.8%
```

> **关键影响**：年电量直接影响日均用电，进而影响所有电池容量推荐和投资回报计算。

## 3. Step 2：月度分布的变化

### 3.1 旧规则

月度分布在 V1.5.0 中**未被使用**。方案推荐不区分月份，只用年电量和24h固定占比。

### 3.2 新规则

引入12个月动态占比，受两个因素影响：
1. **季节乘数**：冷暖季月份的电量占比被放大
2. **热力额外电量按季节权重分配**：供暖季/制冷季月份分配更多热力电量

**澳洲各州冷暖季定义：**

| 州 | 冷季（制冷） | 暖季（供暖） |
|----|-----------|-----------|
| NSW/VIC/SA/WA | Dec–Mar | Jun–Aug |
| QLD | Oct–Apr（长达7个月） | Jun–Jul |
| NT | Sep–Apr | —（无供暖） |
| TAS | Jan–Feb | May–Sep |
| ACT | Dec–Feb | May–Sep |

### 3.3 月度计算公式（完整步骤）

**对每个月 m：**

```
① 季节乘数
SeasonalMult[m] = 1
  + IF(系统含AC或热泵 AND m∈冷季, CoolMonthMult - 1, 0)
  + IF(系统含电采暖或热泵 AND m∈暖季, HeatMonthMult - 1, 0)

② 重塑基准电量
ReshapedBase[m] = 基础年电量 × BaseMonthlyShare[m] × SeasonalMult[m]

③ 归一化回基础年电量
NormBase[m] = ReshapedBase[m] / SUM(ReshapedBase[所有月]) × 基础年电量

④ 热力额外分配（按季节权重）
ThermalWeight[m]:
  - Electric heating：IF(m∈暖季, BaseShare[m] × HeatMonthMult, 0)
  - Heat pump：IF(m∈冷季, BaseShare[m] × CoolMonthMult, 0)
              + IF(m∈暖季, BaseShare[m] × HeatMonthMult, 0)
  - AC / No system：0
ThermalAlloc[m] = 热力额外年电量 × ThermalWeight[m] / SUM(ThermalWeight[所有月])

⑤ EV额外均分
EVAlloc[m] = EV额外年电量 / 12

⑥ 最终月电量
FinalKWh[m] = NormBase[m] + ThermalAlloc[m] + EVAlloc[m]
FinalShare[m] = FinalKWh[m] / 最终年电量
```

### 3.4 月度计算示例：NSW，热泵 Medium

**基础参数：**
- 基础年电量：7,778 kWh
- 系统：Heat pump，使用强度：Medium（CoolMonthMult=1.10，HeatMonthMult=1.15）
- 热力额外：1,800 × 1.0 = 1,800 kWh
- EV：无（0 kWh）
- 最终年电量：9,578 kWh

**NSW 冷季：Dec, Jan, Feb, Mar；暖季：Jun, Jul, Aug**

**以1月（Jan）为例：**
```
BaseShare[Jan] = 0.08548（AU_monthly_share.csv，NSW行）
BaseKWh[Jan] = 7778 × 0.08548 = 665.4 kWh

冷季标志 = 1（Jan∈Dec-Mar）
暖季标志 = 0（Jan∉Jun-Aug）

SeasonalMult[Jan] = 1 + (1.10-1) + 0 = 1.10
ReshapedBase[Jan] = 665.4 × 1.10 = 731.9 kWh
```

**以6月（Jun）为例：**
```
BaseShare[Jun] = 0.10548
BaseKWh[Jun] = 7778 × 0.10548 = 820.4 kWh

冷季标志 = 0（Jun∉Dec-Mar）
暖季标志 = 1（Jun∈Jun-Aug）

SeasonalMult[Jun] = 1 + 0 + (1.15-1) = 1.15
ReshapedBase[Jun] = 820.4 × 1.15 = 943.5 kWh
```

**以5月（May）为例（中性月）：**
```
BaseShare[May] = 0.08466
BaseKWh[May] = 7778 × 0.08466 = 658.7 kWh

SeasonalMult[May] = 1 + 0 + 0 = 1.0（无修正）
ReshapedBase[May] = 658.7 kWh
```

**12个月 ReshapedBase 汇总（简化计算）：**

| 月份 | BaseShare | SeasonalMult | ReshapedBase |
|-----|---------|------------|------------|
| Jan | 0.08548 | 1.10（冷季） | 731.0 |
| Feb | 0.07781 | 1.10（冷季） | 666.4 |
| Mar | 0.07507 | 1.10（冷季） | 642.8 |
| Apr | 0.07137 | 1.00 | 555.0 |
| May | 0.08466 | 1.00 | 658.6 |
| Jun | 0.10548 | 1.15（暖季） | 943.4 |
| Jul | 0.10671 | 1.15（暖季） | 954.4 |
| Aug | 0.09452 | 1.15（暖季） | 845.3 |
| Sep | 0.07356 | 1.00 | 572.0 |
| Oct | 0.07205 | 1.00 | 560.4 |
| Nov | 0.07301 | 1.00 | 567.8 |
| Dec | 0.08027 | 1.10（冷季） | 687.1 |
| **合计** | **1.0000** | | **8,384.2** |

**NormBase（归一化回7778 kWh）：**
```
NormBase[Jan] = 731.0 / 8384.2 × 7778 = 678.3 kWh
NormBase[Jun] = 943.4 / 8384.2 × 7778 = 875.6 kWh
NormBase[May] = 572.0 / 8384.2 × 7778 = 530.6 kWh（之前写错了，取Sep值）
```

**ThermalWeight（热泵：冷+暖季都有）：**
```
ThermalWeight[Jan] = 0.08548 × 1.10（冷季）= 0.09403
ThermalWeight[Jun] = 0.10548 × 1.15（暖季）= 0.12130
ThermalWeight[May] = 0（非冷非暖季）

SUM(ThermalWeight[全年]) = (Jan+Feb+Mar+Dec的冷季权重) + (Jun+Jul+Aug的暖季权重)
= (0.09403+0.08559+0.08258+0.08830) + (0.12130+0.12271+0.10870)
= 0.35050 + 0.35271 = 0.70321
```

**ThermalAlloc（总热力额外1800 kWh按权重分配）：**
```
ThermalAlloc[Jan] = 1800 × 0.09403/0.70321 = 240.7 kWh
ThermalAlloc[Jun] = 1800 × 0.12130/0.70321 = 310.5 kWh
ThermalAlloc[May] = 1800 × 0 / 0.70321 = 0 kWh（中性月无热力分配）
```

**FinalKWh（最终月电量）：**
```
FinalKWh[Jan] = 678.3 + 240.7 + 0 = 919.0 kWh
FinalKWh[Jun] = 875.6 + 310.5 + 0 = 1186.1 kWh
FinalKWh[May] = NormBase[May] + 0 + 0 = ≈530 kWh
```

**与旧版对比：**
```
旧版 NSW 1月：7778 × 0.08548 / 12 ≈ 648 kWh（旧版无月度概念，按均分）
  实际旧版不计算月度，方案不依赖月度分布。

新版 NSW 1月（热泵Medium）：919.0 kWh  ← 明显更高（冷季制冷+分配热力）
新版 NSW 6月（热泵Medium）：1186.1 kWh ← 最高（暖季供暖+分配热力）
新版 NSW 5月：~530 kWh ← 最低（中性月，无额外）
```

> **月度分布的意义**：虽然 V1.5.0 方案推荐不直接使用月度分布，但月度分布是未来账单对账、季节性分析、德国方案推荐的基础。

## 4. Step 3：24h分布的变化

### 4.1 旧规则（V1.5.0固定占比）

各州使用固定的24h分布，不受任何用电行为影响。以NSW为例：

| 时段 | 固定占比 | 固定日均用电（NSW 7778/365=21.3 kWh/天） |
|------|--------|----------------------------------------|
| H07-H16（白天） | 43.88% | 9.35 kWh |
| H17-H20（晚高峰） | 22.1% | 4.71 kWh |
| H17-H06（整夜） | 59.6% | 12.70 kWh |

### 4.2 新规则

24h分布受三个因子叠加影响：

**① 基准小时占比**（来自各州预设，已体现州差异）

**② 居住占用模式修正**（影响白天9-17时段）：
| 模式 | 白天09-17乘数 |
|------|------------|
| Mostly away | 0.6（白天很少用电） |
| Working from home | 1.4（白天大量用电） |
| Someone always home | 1.2（白天适度用电） |

**③ 冷暖峰时段修正**：
| 时段 | 条件 | 乘数（Medium为例） |
|------|------|----------------|
| 14:00-19:00（冷峰） | 系统含AC或热泵 | 1.20 |
| 06:00-09:00 & 18:00-22:00（暖峰） | 系统含电采暖或热泵 | 1.15 |

**计算流程：**
```
AdjShare[h] = BaseHourlyShare[h] × OccMult[h] × PeakMult[h]
NormShare[h] = AdjShare[h] / SUM(AdjShare[所有小时])  ← 归一化

NonEV_kWh[h] = (基础年电量 + 热力额外) / 365 × NormShare[h]
EV_kWh[h] = (EV额外年电量 / 365) × EVChargingDist[h][充电模式]

FinalKWhPerDay[h] = NonEV_kWh[h] + EV_kWh[h]
FinalHourlyShare[h] = FinalKWhPerDay[h] / (最终年电量 / 365)
```

### 4.3 逐小时计算示例：NSW，Working from home + 热泵 Medium

**参数：**

- 占用：WFH → 白天(H09-H16)乘以1.4
- 系统：Heat pump（CoolPeakMult=1.20, HeatPeakMult=1.15）
- 冷峰：H14-H18（NSW制冷季 Dec-Mar 的峰值时段）
- 暖峰：H06-H08 & H18-H21（NSW供暖季 Jun-Aug 的峰值时段）
- EV：无

> **关键变化**：WFH用户白天占比从44%升至50%+，意味着光伏自用率大幅提升，电池需求相应减少。

## 5. Step 4：派生用电需求的变化

这是**澳洲方案推荐最核心的改动点**。

### 5.1 旧规则（固定占比）

```
日均用电 = 7,778 / 365 = 21.3 kWh

白天用电(07-17) = 21.3 × 43.88% = 9.35 kWh    ← 固定
晚高峰用电(17-21) = 21.3 × 22.1% = 4.71 kWh   ← 固定
整夜用电(17-07) = 21.3 × 59.6% = 12.70 kWh    ← 固定
```

**所有 NSW 用户看到的方案基于完全相同的用电数字。**

### 5.2 新规则（动态占比）

```
日均用电 = Final_Annual_kWh / 365    ← 随年电量变化

白天用电(07-17) = 日均 × SUM(FinalHourlyShare[H07..H16])   ← 动态
晚高峰用电(17-21) = 日均 × SUM(FinalHourlyShare[H17..H20]) ← 动态
整夜用电(17-07) = 日均 × SUM(FinalHourlyShare[H17..H23] + FinalHourlyShare[H00..H06]) ← 动态
```

### 5.3 典型场景对比表（基于NSW，不同因子组合）

| 场景 | 年电量 | 日均 | 白天用电 | 晚高峰 | 整夜 | 说明 |
|------|-------|------|--------|-------|------|------|
| **V1.5.0旧版（所有NSW）** | 7,778 | 21.3 | **9.35（44%）** | **4.71（22%）** | **12.70（60%）** | 固定 |
| 全默认（mostly away，no系统） | 7,778 | 21.3 | ~9.3（44%） | ~4.7（22%） | ~12.7（60%） | ≈与旧版一致 |
| WFH，无系统 | 7,778 | 21.3 | **~10.7（50%）** | ~4.5（21%） | ~11.1（52%） | 白天↑ |
| Mostly away + AC Medium | 7,778 | 21.3 | ~9.5（45%） | **~5.3（25%）** | ~12.5（59%） | 下午峰↑ |
| WFH + Heat pump Medium | 9,578 | **26.2** | **~13.2（50%）** | ~6.3（24%） | ~13.7（52%） | 全面↑ |
| Mostly away + EV 10k（夜充） | 9,578 | 26.2 | ~9.5（36%） | ~4.7（18%） | **~16.7（64%）** | 夜间大↑ |
| WFH + EV 10k（太阳能优化充电） | 9,578 | 26.2 | **~14.3（55%）** | ~4.7（18%） | ~12.5（48%） | 白天峰↑ |

> **结论**：不同用电习惯导致白天/夜间占比相差最高可达 **±10-20个百分点**，直接影响光伏剩余和电池容量计算。

## 6. Step 5：方案推荐的联动变化

### 6.1 方案公式本身不变

电池容量、逆变器选型、光伏规格的计算公式**完全不变**。

**变化的是输入数据：**

```
旧版输入：固定的 [日均用电, 白天用电, 晚高峰, 整夜]
新版输入：LoadProfile动态输出的 [日均用电, 白天用电, 晚高峰, 整夜]
```

### 6.2 电池容量影响示意

以NSW VIC 方案B（主流方案）为例，电池容量 ≈ 晚高峰用电 × 某系数：

| 场景 | 晚高峰用电 | 推算电池需求 | 变化 |
|------|---------|-----------|------|
| 旧版NSW | 4.71 kWh | ~10 kWh | 基准 |
| WFH + Heat pump | 6.3 kWh | ~13.5 kWh | **+35%** |
| Mostly away + EV夜充 | 4.7 kWh（晚高峰不变，但整夜大） | 按A方案整夜算更大 | 方案A更适合 |

### 6.3 光伏剩余影响示意

光伏剩余 ≈ 日均发电 − 白天用电

| 场景 | 白天用电 | 光伏剩余（假设发电15kWh/天） | 变化 |
|------|--------|------------------------|------|
| 旧版 mostly away | 9.35 kWh | **5.65 kWh** | 基准 |
| WFH | ~10.7 kWh | **4.3 kWh** | ↓ -24% |
| WFH + EV太阳能充电 | ~14.3 kWh | **0.7 kWh** | ↓ 几乎全自用！ |

> **结论**：WFH + 太阳能充电 EV 的用户光伏几乎全部自用，几乎不需要大电池；而夜间充电EV用户则需要更大电池来存储白天剩余电量供夜间使用。**同一套硬件参数，不同用电习惯下推荐的系统方案可以完全不同。**

## 11. 边界案例与特殊规则说明

### 11.1 默认值兼容性（向后兼容）

当用户未选择任何 LoadProfile 因子，全部使用默认值时：

| 默认值 | 说明 |
|--------|------|
| Occupancy: Mostly away | 白天乘数=0.6，接近固定预设的白天占比 |
| HVAC: No system | 不增加额外电量，不修改季节分布 |
| Usage: Medium | 系数全为1（×1.0倍率，不影响结果） |
| EV: No vehicle | EV额外=0 |
| Charging: Mostly overnight | 无EV时此因子无效 |

**结果**：年电量 = 基准，24h分布 ≈ 州预设固定值。**与V1.5.0基本一致**，向后兼容。

### 11.2 AC的特殊性

Air conditioning 系统：
- **不增加年电量**（基础热负荷=0）
- **只改变月度和24h分布**（冷季月份和冷峰时段占比上升）
- 用电强度Medium → 冷季月乘数1.10，冷峰乘数1.20
- 实际效果：把已有年电量的更多份额"搬移"到夏季高峰时段

### 11.3 季节标志叠加规则

当系统为 Heat pump（冷暖都管）：
- 同时激活冷季月份修正 + 暖季月份修正
- 同时激活冷峰时段修正 + 暖峰时段修正
- 若某小时同时处于冷峰和暖峰（如H18在德国），两个乘数相乘：1.20 × 1.15 = 1.38

### 11.4 QLD 的特殊处理

QLD 制冷季长达7个月（Oct-Apr），供暖季仅2个月（Jun-Jul）：
- 热力权重的绝大部分分配给冷季（7个月）
- 使用 Very high 时冷季月乘数高达1.35，月度曲线极度向夏季倾斜
- 这与QLD实际用电曲线（夏季12-2月最高）相符

### 11.5 NT 的特殊处理

NT（北领地）：
- 冷季（制冷）：Sep-Apr（8个月）
- 暖季（供暖）：**无**（热带气候，几乎不需要供暖）
- 热泵选择时：ThermalWeight暖季=0，所有热力额外分配到冷季

### 11.6 TAS/ACT 的较高基准年电量

TAS（10,148 kWh）和ACT（8,632 kWh）基准年电量远高于其他州，叠加 Heat pump + Very high 后年电量可超 14,000 kWh，方案推荐需要明显更大的PV+电池系统。
